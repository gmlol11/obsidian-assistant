# Участие в разработке

## Перед началом

1. Прочитайте [контекст проекта](docs/project-context.md), [архитектуру](docs/architecture.md) и [модель угроз](docs/threat-model.md).
2. Проверьте открытые задачи и ADR.
3. Для изменения модели безопасности сначала предложите новый ADR.

## Локальная проверка

```bash
make check
```

Тесты должны использовать только временные каталоги или `tests/fixtures/vault`. Не используйте копию личного хранилища в тестах и не добавляйте реальные заметки в фикстуры.

## Pull request

- делайте одно логическое изменение на PR;
- опишите пользовательский эффект и риски;
- добавьте или обновите тесты;
- обновите документацию и `[Unreleased]` в `CHANGELOG.md`;
- явно укажите, затрагивает ли изменение запись в vault, секреты, сеть или права OpenClaw.

Коммитам рекомендуется формат `type(scope): summary`, например `feat(capture): add Telegram text intake`.
