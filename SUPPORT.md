# Поддержка

Для ошибок и предложений используйте GitHub Issues с подходящим шаблоном. Перед созданием задачи выполните `make check` и приложите обезличенный вывод.

Не публикуйте токены, содержимое личных заметок, абсолютные домашние пути, адрес Mac mini или внутренние сведения MOEX.

Уязвимости отправляйте по инструкции из [SECURITY.md](SECURITY.md), а не через публичный issue.
