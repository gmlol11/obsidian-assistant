# ADR-0002: Отделить агента от компонента записи в vault

- Статус: accepted
- Дата: 2026-07-16

## Контекст

OpenClaw взаимодействует с внешними сообщениями, моделью и потенциально недоверенным контентом. Прямой доступ такого компонента ко всему хранилищу и shell создаёт слишком большой радиус ошибки.

## Решение

OpenClaw выполняет диалог и оркестрацию, но передаёт запись отдельному Vault Worker через ограниченный версионированный контракт. Worker не исполняет произвольные команды и повторно проверяет каждое поле запроса.

## Альтернативы

- дать OpenClaw прямой доступ к vault — проще, но рискованнее;
- встроить Telegram, LLM и файловую запись в один процесс — меньше компонентов, но хуже изоляция и тестируемость.

## Последствия

- появляется дополнительный интерфейс и очередь;
- сетевой агент можно обновлять независимо;
- файловая политика остаётся небольшой и проверяемой;
- компрометация диалога не должна автоматически означать полный доступ к диску.
