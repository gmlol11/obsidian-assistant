# ADR-0003: Использовать create-only, allowlist и dry-run

- Статус: accepted
- Дата: 2026-07-16

## Контекст

Главный риск ранней автоматизации — незаметно повредить существующие заметки или записать данные не в тот каталог.

## Решение

Базовая политика состоит из трёх независимых ограничений:

1. `dry-run` включён по умолчанию;
2. запись разрешена только в перечисленные относительные каталоги;
3. операция создаёт новый файл эксклюзивно и не перезаписывает существующий.

Редактирование, перемещение и удаление не входят в базовый контракт и требуют отдельных решений.

## Последствия

- первые версии могут создавать больше входящих файлов;
- случайное повторение не уничтожает данные;
- для обновления проектных заметок понадобится отдельный контролируемый механизм;
- тесты политики являются обязательными для каждого нового типа операции.
