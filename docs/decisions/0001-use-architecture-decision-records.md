# ADR-0001: Использовать Architecture Decision Records

- Статус: accepted
- Дата: 2026-07-16

## Контекст

Проект должен долго развиваться и переноситься между компьютерами. Причины ограничений безопасности и инфраструктурных решений нельзя надёжно восстановить только из итогового кода.

## Решение

Существенные решения фиксируются отдельными неизменяемыми ADR в `docs/decisions`. Изменение принятого решения создаёт новый ADR и связывает его с предыдущим.

## Последствия

- будущий разработчик видит причины решения;
- документация требует небольшой дисциплины;
- мелкие локальные изменения не нуждаются в ADR.
