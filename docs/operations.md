# Эксплуатация

## Цели надёжности

Для персонального сервиса целевые показатели являются ориентирами:

- входящий текст подтверждается не позднее одной минуты при доступном Telegram;
- сырой материал сохраняется даже при недоступной LLM;
- ежедневный обзор приходит в заданном временном окне;
- любое отклонённое действие имеет безопасную причину и идентификатор;
- система не теряет входящие молча.

## Ежедневная проверка

- сервисы запущены;
- очередь не растёт бесконтрольно;
- нет элементов в карантине;
- последний backup завершён;
- на диске достаточно места;
- ежедневный обзор отправлен;
- не было отказов политики путей, требующих расследования.

## Журналирование

Разрешено записывать:

- timestamp;
- request ID;
- тип операции;
- обезличенный actor ID или его хеш;
- относительный целевой раздел;
- результат и код ошибки;
- длительность.

Не записывать:

- токены и заголовки авторизации;
- полный текст заметок и prompt;
- вложения;
- абсолютный домашний путь;
- ответы модели, содержащие исходный пользовательский текст.

## Обновление

1. Проверить changelog зависимости и security advisories.
2. Создать backup.
3. Запустить CI и smoke test на фикстуре.
4. Развернуть версию в dry-run.
5. Проверить один контролируемый capture.
6. Разрешить рабочую запись.
7. Записать версию и результат в приватный журнал эксплуатации.

## Резервное копирование

Минимум две независимые копии: локальная история/Time Machine и отдельная зашифрованная копия. Obsidian Sync не заменяет backup. Порядок проверки описан в [runbook](runbooks/backup-and-restore.md).

## Сбои

- **LLM недоступна:** сохранить raw capture, классификацию отложить.
- **Telegram недоступен:** очередь продолжает обработку локальных задач; уведомление отправить после восстановления.
- **Vault недоступен:** ничего не создавать в альтернативном случайном пути; запрос поместить в карантин.
- **Политика отклонила путь:** не обходить защиту автоматически; расследовать входящий payload.
- **Недостаточно места:** остановить запись, сохранить очередь и уведомить владельца.
- **Подозрение на компрометацию:** выполнить [incident response](runbooks/incident-response.md).
