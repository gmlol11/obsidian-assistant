# Модель угроз

## Защищаемые активы

- содержимое Obsidian vault;
- API-ключи и Telegram token;
- контроль над OpenClaw и Mac mini;
- целостность задач, напоминаний и ежедневных обзоров;
- резервные копии и история изменений.

## Потенциальные источники угроз

- посторонний пользователь Telegram;
- вредоносный текст, файл или prompt injection;
- ошибочный ответ модели;
- непроверенный OpenClaw skill или обновление зависимости;
- неверная конфигурация путей и сетевого доступа;
- случайная ошибка владельца;
- компрометация домашнего устройства или токена.

## Основные угрозы и меры

| Угроза | Влияние | Основные меры |
|---|---|---|
| Посторонний отправитель вызывает инструменты | Несанкционированные действия | allowlist Telegram ID, pairing, отдельная доверенная граница |
| Подмена канала или sender ID в плагине | Несанкционированный Capture | доверенная авторизация OpenClaw плюс повторная проверка channel и `ownerIds` |
| Prompt injection просит выполнить shell | Компрометация Mac mini | shell запрещён по умолчанию, структурированный контракт операций |
| Path traversal или симлинк | Запись вне vault | canonical path, запрет `..`, проверка корня и allowlist |
| Модель выбирает опасную папку | Повреждение данных | модель только предлагает, политика повторно валидирует |
| Повторная доставка события | Дубликаты | request ID и идемпотентность очереди |
| Подмена файла очереди | Запись изменённого текста | SHA-256 канонического события, строгая схема, карантин |
| Два worker одновременно | Повторная обработка | файловая блокировка, processing lease, один сервер |
| Случайная перезапись | Потеря заметки | create-only, exclusive create, version history и backup |
| Утечка через логи | Раскрытие заметок и токенов | редактирование секретов, metadata-only logging |
| Публичный Gateway | Удалённый контроль | loopback bind, auth token, приватный удалённый доступ |
| Вредоносный plugin/skill | Выполнение кода | allowlist, ручная проверка исходников и закрепление версии |
| Утечка token в Vault Worker | Расширение последствий компрометации | отдельные OpenClaw и worker env, очищенное окружение дочернего процесса |
| Конкурирующие sync-клиенты | Конфликты и потеря изменений | один механизм синхронизации на устройство |
| Потеря питания | Недоступность сервиса | UPS, runbook, уведомление, ручная разблокировка FileVault |

## Остаточные риски

- доверенный владелец может явно разрешить опасную операцию;
- локальный администратор Mac mini имеет доступ к процессам и данным;
- pending, processing и quarantine содержат исходный текст и требуют защиты диска и прав доступа;
- облачная модель обрабатывает переданный ей текст в соответствии со своей политикой;
- уязвимость в OpenClaw или зависимости может обойти прикладные ограничения;
- резервная копия бесполезна, если восстановление никогда не проверялось.

## Обязательные проверки перед расширением прав

- тест постороннего Telegram ID;
- тест выхода через `..`, абсолютный путь и симлинк;
- тест ответа модели с инструкцией обойти политику;
- проверка Gateway с другого устройства в сети;
- проверка редактирования секретов в логах;
- восстановление backup в отдельный каталог;
- OpenClaw security audit после изменения конфигурации;
- проверка, что установленный Capture plugin собран против закреплённого диапазона OpenClaw.
