# Политика безопасности

## Поддерживаемые версии

До первого публичного релиза поддерживается только ветка `main`.

## Сообщение об уязвимости

Не создавайте публичный issue для уязвимостей, утечек секретов или способов получить доступ за пределы разрешённого vault. Используйте приватный GitHub Security Advisory репозитория.

В сообщении укажите:

- затронутую версию или commit;
- ожидаемое и фактическое поведение;
- минимальный сценарий воспроизведения без реальных секретов и заметок;
- возможное влияние;
- предложенное исправление, если оно известно.

## Критические категории

- выход за пределы корня vault;
- обход allowlist папок;
- перезапись или удаление заметок без подтверждения;
- исполнение произвольных команд через входящее сообщение;
- раскрытие токенов, ключей или текста заметок в логах;
- раскрытие payload из pending, processing или quarantine очереди;
- доступ постороннего отправителя к агенту;
- публично доступный OpenClaw Gateway без строгой аутентификации.

При подозрении на компрометацию следуйте [плану реагирования](docs/runbooks/incident-response.md).
